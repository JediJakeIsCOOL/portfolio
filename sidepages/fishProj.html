Fish Website

Project Summary

This fish website was made using HTML, CSS, JavaScript, Node.js, and EJS for templating. The EJS templating allowed me to create multiple pages with the same outline but different information and images on each one of them. The projects purpose was to get some backend development experience. I used node.js and the express library in order to create a configuration page for the app. JSON files were used as the placeholder database. Then we used Embedded JavaScript(ejs) in order to create templating pages. The chat and the feedback page that was implemented into the website was made using the socket.io library module.

Interesting Code Used

This is the configuation page and the beauty of Node.JS and the express library. This configuration page allows me to load all the modules and libaries I need for the application I am creating. It also links me to each subsequent page that will be created for the application. The local server is created and then I was able to see changes on our web app through localhosting. Using the socket.io library I was able to make a direct communication line from the front-end to the back-end, allowing chat features.

Each of the pages of my web application has a route page. This is an example of one of my routes for a page that displayed bluegill information. I created a function to update my JSON database with a forloop. If the data was labeled a specific name, then the fields would be populated with the specific fish's information. These routes take advantage of the express libaries .router method. This allows us to link this page to the main configuration page.
